# Give automatic compensation for late orders

## Цель проекта

Проанализировать сколько компенсаций будет выдавать бот(проактивно) при каждом опоздании заказа.

## Данные

Датасет заказов.

Количество записей: 8 898 659 заказов.

## Инструменты

SQL  
GreenPlum 

## Анализ

В ходе анализа была построена таблица с 1.10.2025 по 31.01.2026:

1. кол-во опоздавших заказов 
2. кол-во обращений в поддержку, по опозданиям заказов
3. кол-во выданных операторами компенсаций
4. кол-во и % использованных компенсаций, которые выдали операторы, за опоздания заказа

За выбранный период:
1. обращалось в поддержку 0,6% клиентов от всех опозданий заказов
2. выдавалось компенсаций 0,2% от всех опозданий заказов
3. использовалось 43,6% от всех выданных компенсаций
4. средняя сумма использованных компенсаций 223 руб

## Выводы

- сейчас на компенсации за опоздания заказов, компения тратит в среднем 675 599,06 руб/мес
- если выдавать проактивно компенсации за опоздания заказов, то затраты компании на такие компенсации могут составить 219 65 331,3. А это в 325 раз больше, чем выдается сейчас. 
Проект убыточный, от него следует отказаться
